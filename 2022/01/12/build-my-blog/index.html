    Git的使用

    init

    将指定目录作为Git仓库，如果为空则为Git Bash的当前目录

    add，commit

    以上命令将目录下以 .c 结所有文件及 README 文件提交到仓库中。

    add

    添加一个或多个文件到暂存区： 添加指定目录到暂存区，包括子目录： 添加当前目录下的所有文件到暂存区： git add . ：他会监控工作区的状态树，使用它会把工作时的所有变化提交到暂存区，包括文件内容修改(modified)以及新文件(new)，但不包括被删除的文件。

git add -u ：他仅监控已经被add的文件（即tracked file），他会将被修改的文件提交到暂存区。add -u 不会提交新文件（untracked file）。（git add –update的缩写）

git add -A ：是上面两个功能的合集（git add –all的缩写）

一般就用git add -A

commit

一般使用 -m 参数

-a 参数设置修改文件后不需要执行 git add 命令，直接来提交

clone

使用 git clone 从现有 Git 仓库中拷贝项目

参数说明：

  • repo: Git仓库，格式为git@github.com:github用户名/项目名.git
  • directory: 本地目录。

    reset

    一般用 soft，谨慎使用 hard 参数，它会删除回退点之前的所有信息。
